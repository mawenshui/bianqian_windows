## 二、紧急修复（P0）

### 2.1 window_positions.json 重复 key 修复 ✅
- **问题**：文件可能包含重复 key，JSON 标准不允许，加载时后值覆盖前值
- **影响**：可能导致窗口位置丢失
- **修复**：
  - `save_position_history()` 改用原子写入（先写临时文件再 `os.replace`）并显式去重
  - `load_position_history()` 增加数据结构校验，跳过无效条目并自动回写清理
  - 损坏文件自动备份为 `.bak` 并重置
- **状态**：✅ 已修复

### 2.2 全局快捷键热键消息检测 ✅
- **问题**：原 `check_messages()` 为空实现，热键注册后检测不到按键
- **修复**：已重写为 `HotkeyNativeEventFilter(QAbstractNativeEventFilter)`，拦截 `WM_HOTKEY` 消息并转发到 `GlobalShortcutManager.handle_hotkey_message()`
- **状态**：✅ 已修复

### 2.3 UndoRedoManager 完整集成 ✅
- **问题**：`UndoRedoManager.save_current_state()` 中 `max_history` 达到上限时 `pop(0)` 后错误地执行 `current_index += 1`，导致索引越界
- **修复**：当历史记录达到上限时 `pop(0)` 后不再递增 `current_index`，因为前移操作已自然保持索引正确
- **状态**：✅ 已修复

### 2.4 自动更新签名验证
- **问题**：`features/updater.py` 下载更新包后直接执行 `.bat` 脚本解压替换，没有对下载文件进行任何签名验证或完整性校验（无 checksum、无代码签名、ghproxy.com 镜像可能被中间人攻击）
- **方案**：
  1. 在 GitHub Release 中发布 SHA256 校验和文件
  2. 下载后验证文件哈希
  3. 对 `.bat` 更新脚本进行完整性检查
- **预估**：2-3 天

### 2.5 WebDAV 密码加密存储
- **问题**：`features/sync_dialog.py` 中 WebDAV 密码直接以明文存储在 `settings.json` 中，虽然配置中有 `password_encrypted` 字段，但同步对话框读写的是明文密码
- **方案**：
  1. 使用 `NoteEncryption` 模块加密 WebDAV 密码
  2. 在 `config.py` 中提供加密/解密封装方法
  3. 确保密码从不以明文形式写入磁盘
- **预估**：1 天

## 三、代码质量优化（P1）

### 3.1 统一日志系统 ✅ 已完成
- **问题**：当前使用 `print()` 和 `QMessageBox` 混合输出
- **方案**：引入 Python `logging` 模块，分级输出
  - DEBUG: 调试信息（文件）
  - INFO: 正常运行日志（文件）
  - WARNING: 非致命错误（文件 + 控制台）
  - ERROR: 用户可见提示（QMessageBox + 文件）
- **实现**：`core/logger.py` — RotatingFileHandler（5MB 上限，3 个备份）
- **状态**：✅ 已完成

### 3.2 类型注解完善
- **问题**：代码中缺少类型提示，大量使用 `hasattr` 运行时检查（如 `hasattr(self, 'rich_text')`、`hasattr(self, 'bold_btn')`、`hasattr(self, 'plugin_loader')`）
- **方案**：
  - 为核心类和方法添加 Python type hints
  - 使用 Protocol 或 ABC 定义接口，通过类型标注替代运行时 hasattr 检查
- **状态**：部分完成（config.py、manager.py 已添加）

### 3.3 单元测试覆盖 ✅ 已完成
- **状态**：✅ 已完成（v1.6.3）
- **实现**：
  - 新增 `tests/test_note.py` — 31 个测试（PlainLineEdit / PlainTextEdit / StickyNote 初始化 / 路径穿越防护 / 撤销重做按钮 / 屏幕几何缓存 / paintEvent 缓存）
  - 新增 `tests/test_tag.py` — 18 个测试（TagManager CRUD / 颜色 / 持久化）
  - 新增 `tests/test_errors.py` — 7 个测试（异常层次结构 / handle_error）
  - 新增 `tests/test_search.py` — 7 个测试（相关度排序 / 搜索索引）
  - 新增 `tests/test_undo_redo.py` — 7 个测试（撤销重做状态 / 信号）
  - 全量测试：222 passed, 1 skipped

### 3.4 配置持久化重构 ✅ 已完成
- **问题**：`settings.json` 读写分散在各处
- **方案**：引入统一的 `ConfigManager` 单例，集中管理所有配置项
- **实现**：`core/config.py` — 原子写入、嵌套键路径访问、变更信号
- **状态**：✅ 已完成

### 3.5 模块解耦
- **问题**：模块间存在不合理的直接依赖
  - `core/note.py` 直接调用 `self.manager.plugin_loader.dispatch_event()`
  - `core/note.py` 的 save_note 中直接 import 并调用 `get_note_cache()`
  - `core/note.py` 直接访问 `self.manager.link_manager`
  - `core/note.py` 的 delete_note 中直接调用 `get_note_cache().invalidate()`
- **方案**：
  1. 使用依赖注入或事件总线解耦
  2. feature 模块不应直接 import 其他 feature 模块的内部实现
  3. 通过信号/槽通信替代直接方法调用
- **预估**：3-4 天

### 3.6 错误处理规范化 ✅ 已完成
- **状态**：✅ 已完成（v1.6.3）
- **实现**：
  - 新建 `core/errors.py` — 统一异常层次结构（StickyNoteError / DataError / FileOperationError / PluginError / SearchError）
  - `core/note.py` — ~10 处 `except Exception: pass` 改为 `logger.debug()` 日志记录
  - `core/manager.py` — ~5 处静默吞掉改为日志记录
  - `features/search.py` — 统一使用 SearchError
- **问题**：代码中存在 4 种不一致的错误处理模式：
  - 静默吞掉异常（`except Exception: pass`）
  - 只打日志（`except Exception as e: logger.error(...)`）
  - 弹窗提示（`except Exception as e: QMessageBox.warning(...)`）
  - 重新抛出（`except Exception as e: raise RuntimeError(...)`）
- **方案**：
  1. 定义错误处理策略：UI 层弹窗、业务层日志、底层 raise
  2. 创建统一的异常层次结构
  3. 为用户可见的错误添加友好的错误消息
- **预估**：2 天

### 7.1 数据加密存储 ✅ 已完成

- **状态**：✅ 已完成

### 7.2 安全删除

### 7.3 备份加密
- **需求**：备份文件支持密码加密
- **预估**：1 天

### 7.4 加密 AAD 增强
- **问题**：`features/encryption.py` 加密时使用 `None` 作为 AAD（附加认证数据），密文可被替换攻击
- **方案**：使用便签 ID 或标题作为 AAD，防止密文被替换攻击
- **预估**：0.5 天

### 7.5 主密码暴力破解防护
- **问题**：`main.py` 最多允许 3 次尝试后退出应用，但无渐进式延迟、无锁定机制（用户可重启应用继续尝试）、无日志记录失败尝试
- **方案**：
  1. 实现指数退避延迟（第 2 次等 1 秒，第 3 次等 5 秒）
  2. 在配置中记录失败次数，启动时检查并实施临时锁定
  3. 记录失败日志用于安全审计
- **预估**：1 天

### 7.6 路径穿越防护 ✅ 已完成
- **状态**：✅ 已完成（v1.6.3）
- **实现**：`core/note.py` __init__ — `os.path.realpath()` 验证 note_file 在 notes_dir 内；`core/manager.py` load_notes — 验证文件路径
- **问题**：`core/note.py` 便签文件路径直接拼接 note_id（`f'note_{self.note_id}.json'`），虽然当前 note_id 是整数，但若未来改为字符串标识符可能产生路径穿越风险
- **方案**：对文件路径进行规范化验证，确保最终路径在 notes_dir 内
- **预估**：0.5 天
