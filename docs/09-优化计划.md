## 二、紧急修复（P0）

### 2.1 window_positions.json 重复 key 修复
- **问题**：文件可能包含重复 key，JSON 标准不允许，加载时后值覆盖前值
- **影响**：可能导致窗口位置丢失
- **修复**：
  - 修复 `save_position_history()` 逻辑，确保不重复写入
  - 清理现有文件中的重复 key
- **预估**：0.5 天

### 2.2 全局快捷键热键消息检测
- **问题**：`GlobalShortcutManager.check_messages()` 为空实现（`pass`），热键注册后检测不到按键
- **影响**：`Ctrl+Shift+N/F/B` 全局快捷键可能不工作
- **修复**：
  - 重写 `check_messages()` 使用 `win32gui.PumpWaitingMessages` 或 `QAbstractNativeEventFilter`
  - 添加 `WM_HOTKEY` 消息处理
- **预估**：2 天

### 2.3 UndoRedoManager 完整集成
- **问题**：`UndoRedoManager` 已实现但未在所有入口完整集成
- **修复**：确保 `StickyNote.initUI()` 中正确创建并绑定 manager 实例
- **预估**：0.5 天

## 三、代码质量优化（P1）

### 3.1 统一日志系统
- **问题**：当前使用 `print()` 和 `QMessageBox` 混合输出
- **方案**：引入 Python `logging` 模块，分级输出
  - DEBUG: 调试信息（文件）
  - INFO: 正常运行日志（文件）
  - WARNING: 非致命错误（文件 + 控制台）
  - ERROR: 用户可见提示（QMessageBox + 文件）
- **预估**：1 天

### 3.2 类型注解完善
- **问题**：代码中缺少类型提示
- **方案**：为核心类和方法添加 Python type hints
- **预估**：2 天

### 3.3 单元测试覆盖
- **问题**：当前测试覆盖不完整
- **方案**：
  - 为 `core/note.py`、`core/manager.py` 添加单元测试
  - 为各 `features/` 模块补充边界用例
  - 集成测试：模拟完整 CRUD + 保存/加载流程
- **预估**：3 天

### 3.4 配置持久化重构
- **问题**：`settings.json` 读写分散在各处
- **方案**：引入统一的 `ConfigManager` 单例，集中管理所有配置项
- **预估**：1 天

## 四、功能增强（P2）

### 4.1 富文本工具栏增强
- **描述**：扩展文字格式能力，接近轻量笔记软件水平
- **需求**：
  - 添加下划线、删除线、上标/下标
  - 文本对齐（左/中/右）
  - 有序列表、无序列表
  - 文本高亮（背景色）
  - 插入链接、插入图片
- **预估**：5 天

### 4.2 便签锁定/密码保护
- **描述**：为单个便签或整个应用添加密码保护
- **需求**：
  - 单便签锁定：右键菜单「锁定」，需输入密码查看内容
  - 应用级密码锁：启动时要求输入主密码
  - 加密存储：AES-256 加密便签内容
- **预估**：5 天

### 4.3 Markdown 渲染模式
- **描述**：支持 Markdown 语法实时预览
- **需求**：
  - 编辑/预览双模式切换
  - 支持标题、列表、代码块、表格、图片
  - 可导出为 PDF
- **预估**：5 天

### 4.4 便签关联与链接
- **描述**：支持便签间互相引用
- **需求**：
  - `[[便签名]]` 语法创建内部链接
  - 点击链接跳转到目标便签
  - 反向链接面板：显示哪些便签引用了当前便签
- **预估**：3 天

### 4.5 云同步（实验性）
- **描述**：支持将便签数据同步到云端
- **方案**：
  - 支持 WebDAV / OneDrive / 坚果云
  - 双向同步 + 冲突解决
  - 离线时缓存变更，联网后自动同步
- **预估**：8 天

### 4.6 插件系统（实验性）
- **描述**：开放插件接口，支持用户扩展功能
- **方案**：
  - 定义 `PluginBase` 接口（on_load, on_unload, register_menu）
  - 插件目录 `plugins/` 自动扫描加载
  - 示例插件：字数统计、番茄钟、天气显示
- **预估**：5 天

## 六、性能优化（P3）

### 6.1 文件 I/O 完全异步化
- **问题**：部分文件操作（备份、导出）仍在主线程同步执行
- **方案**：全部使用 `QThread` 异步执行
- **预估**：2 天

### 6.2 便签数据缓存
- **问题**：搜索时重复读取未打开便签的 JSON 文件
- **方案**：实现 LRU 缓存，内存中保留最近访问的便签数据
- **预估**：1 天

### 6.3 启动速度优化
- **问题**：启动时加载所有模块
- **方案**：实现延迟加载（LazyLoader），按需初始化 features 模块
- **预估**：1 天

### 6.4 内存优化
- **问题**：大量便签时内存占用较高
- **方案**：
  - 仅加载可见便签的完整数据
  - 隐藏便签仅保留元数据（标题、位置、标签）
  - 定期清理未使用的 Qt 资源
- **预估**：2 天
