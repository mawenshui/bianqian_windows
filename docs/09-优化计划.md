## 二、紧急修复（P0）

### 2.1 window_positions.json 重复 key 修复
- **问题**：文件包含两个 `"1"` key，JSON 标准不允许重复 key，加载时后值覆盖前值
- **影响**：可能导致窗口位置丢失
- **修复**：
  - 修复 `save_position_history()` 逻辑，确保不重复写入
  - 清理现有文件中的重复 key
- **预估**：0.5 天

### 2.2 全局快捷键热键消息检测缺失
- **问题**：`GlobalShortcutManager.check_messages()` 为空实现（`pass`），热键注册后实际检测不到按键
- **影响**：`Ctrl+Shift+N/F/B` 等全局快捷键可能不工作
- **修复**：
  - 重写 `check_messages()` 使用 `win32gui.PumpWaitingMessages` 或替代方案
  - 或使用 `QAbstractNativeEventFilter` 实现 Windows 消息过滤
  - 添加 Win32 `WM_HOTKEY` 消息处理
- **预估**：2 天

### 2.3 UndoRedoManager 未集成到 StickyNote
- **问题**：`features/undo_redo.py` 中 `UndoRedoManager` 类已完整实现，但 `StickyNote` 中未创建其实例，也未调用 `set_undo_redo_manager()`
- **影响**：虽然 `UndoRedoTextEdit`/`UndoRedoLineEdit` 依赖 manager 处理 Ctrl+Z/Y，但 manager 为 None 时快捷键不会触发 undo_redo 逻辑（目前可能回退到 Qt 内置支持）
- **修复**：
  - 在 `StickyNote.initUI()` 中创建 `UndoRedoManager` 实例
  - 调用 `title_edit.set_undo_redo_manager()` 和 `text_edit.set_undo_redo_manager()`
- **预估**：1 天

## 三、代码质量优化（P1）

### 3.1 main.py 模块化拆分
- **问题**：`main.py` 单文件 1802 行，包含 `StickyNote`、`StickyNoteManager`、`SettingsDialog`、`PlainLineEdit`、`PlainTextEdit` 五个类
- **目标**：拆分为 `core/` 模块
- **方案**：
  ```
  core/
  ├── note.py          # StickyNote, PlainLineEdit, PlainTextEdit
  ├── manager.py       # StickyNoteManager
  └── settings.py      # SettingsDialog
  main.py              # 仅保留 main() 入口 + 导入
  ```
- **预估**：3 天

### 3.2 统一日志系统
- **问题**：当前使用 `print()` 和 `QMessageBox` 混合输出
- **方案**：引入 Python `logging` 模块，分级输出
  - DEBUG: 调试信息（文件）
  - INFO: 正常运行日志（文件）
  - WARNING: 非致命错误（文件 + 控制台）
  - ERROR: 用户可见提示（QMessageBox + 文件）
- **预估**：1 天

### 3.3 类型注解补充
- **问题**：代码中缺少类型提示
- **方案**：为核心类和方法添加 Python type hints
  ```python
  def set_font(self, font_settings: dict) -> None:
  def load_notes(self) -> None:
  ```
- **预估**：2 天

## 四、功能增强（P2）

### 4.1 定时提醒系统 ⭐
- **描述**：为便签添加定时提醒功能
- **需求**：
  - 支持一次性提醒和周期提醒（每天/每周/每月）
  - 到期时弹出系统通知
  - 可选择是否自动打开便签
  - 提醒数据持久化存储
- **技术**：`QTimer` 定时轮询 + `QSystemTrayIcon.showMessage()`
- **预估**：5 天

### 4.2 标签分组管理 ⭐
- **描述**：为便签添加标签，支持按标签分组查看
- **需求**：
  - 每个便签可添加多个标签
  - 托盘菜单按标签分组显示
  - 搜索支持按标签过滤
  - 标签颜色自定义
- **技术**：`TagManager` 类 + `tags.json` 存储
- **预估**：4 天

### 4.3 导入导出功能
- **描述**：支持便签数据的外部导入导出
- **需求**：
  - 导出为 TXT/Markdown 格式
  - 导入 TXT 内容创建新便签
  - 批量导入导出
- **预估**：3 天

### 4.4 便签模板系统
- **描述**：预定义便签模板，快速创建特定类型便签
- **需求**：
  - 内置模板：待办清单、会议纪要、周计划
  - 用户自定义模板
  - 模板管理界面
- **预估**：3 天

### 4.5 历史版本管理
- **描述**：为每个便签保留编辑历史
- **需求**：
  - 自动快照保存
  - 历史版本时间线查看
  - 选择版本恢复
  - 版本对比（diff）
- **预估**：5 天

## 六、性能优化（P3）

### 6.1 文件 I/O 异步化
- **问题**：`save_note()` 等文件操作在主线程同步执行
- **方案**：使用 `QThread` 或 `asyncio` 异步写入
- **预估**：2 天

### 6.2 便签数据缓存
- **问题**：搜索时重复读取未打开便签的 JSON 文件
- **方案**：实现 LRU 缓存，内存中保留最近访问的便签数据
- **预估**：1 天

### 6.3 启动速度优化
- **问题**：启动时加载所有模块
- **方案**：实现延迟加载（LazyLoader），按需初始化 features 模块
- **预估**：1 天
